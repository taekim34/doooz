# DOOOZ 마법의 프롬프트 — 단계별 세트

> 아이와 부모가 함께 AI 채팅에 하나씩 입력하며 앱을 만들어 가는 가이드입니다.
> 각 단계에서 AI가 코드를 만들어주면, 따라하고 다음 프롬프트를 입력하세요.
> 
> **💡 팁:**
> 
> - 프롬프트를 그대로 복사해도 되고, 자기만의 아이디어로 바꿔도 돼요!
> - 에러가 나면 "이런 에러가 났어: [에러 메시지]" 라고 AI에게 보내주세요.
> - AI가 이상한 코드를 주면 "다시 확인해줘" 라고 하면 돼요.

---

## 시작하기 전에

AI에게 첫 번째 프롬프트를 보내기 전에, 아래 준비물이 필요해요:

1. **컴퓨터** (Windows, Mac, 또는 Linux)
2. **AI 채팅** (Claude, ChatGPT, Gemini 등 아무거나)
3. **터미널** (명령어를 입력하는 검은 화면) — 모르면 "내 컴퓨터에서 터미널 여는 법"을 구글에 검색!
4. **Node.js** — 모르면 "Node.js 설치 방법"을 구글에 검색!

준비됐으면 시작!

## 프롬프트 #1 — 우리 앱을 소개하기 🎬

```
안녕! 나는 우리 가족이 같이 쓸 "할일 앱"을 만들고 싶어.

이런 앱이야:
- 이름은 "DOOOZ" (두우즈)
- 부모가 아이한테 할일을 줘 (예: 양치하기, 책읽기, 방 정리)
- 아이가 할일을 끝내면 체크! → 포인트를 받아
- 포인트를 모으면 보상이랑 바꿀 수 있어 (예: 아이스크림, 게임 30분)
- 레벨이 올라가고 (30단계!), 배지도 모을 수 있어 (60개!)
- 캐릭터가 성장해: 🐣병아리 → 🧒루키 → ⚔️용사 → 🦸영웅 → 👑전설
- "조르기" 기능: 아이가 "나 이것도 했어!" 하고 추가 포인트를 요청
- "봐주기" 기능: 바쁜 날 못한 할일은 부모가 벌점 없이 면제
- 매일 저녁에 "아직 안 한 할일 있어!" 알림이 와
- 한국어, 영어, 일본어 3개 언어 지원
- 스마트폰에 앱처럼 설치할 수 있어

기술은 Next.js + React + TypeScript + Tailwind + Supabase + Vercel 로 만들어줘.

이해했으면 "준비됐어!" 라고 해줘. 그 다음부터 하나씩 같이 만들자!
```

## 프롬프트 #3 — 데이터베이스 만들기 💾

> **부모님께:** 이 단계에서는 [Supabase](https://supabase.com)에 무료 가입 후 프로젝트를 만들어야 해요. 좌측 메뉴 Project Settings → API Keys에서 URL과 키를 확인하세요.

```
2단계야!

Supabase 데이터베이스를 만들어줘. 우리 앱에 필요한 것들:

가족 관련:
- 가족 정보 (가족이름, 초대코드, 시간대, 언어)
- 가족 구성원 (부모/아이 구분, 캐릭터, 포인트, 레벨)

할일 관련:
- 할일 템플릿 (매주 반복하는 것 / 한번만 하는 것)
- 매일 할일 (오늘 할 것들, 완료했는지 여부)
- "조르기"로 요청한 할일도 여기에 포함

포인트와 보상:
- 포인트 기록 (절대 수정/삭제 안 되는 가계부처럼!)
- 보상 목록과 교환 요청

성장 시스템:
- 배지 60개 (할일 달성, 연속 출석, 완벽한 하루 등)
- 캐릭터 12종

보안:
- 다른 가족의 데이터는 절대 볼 수 없게 보호
- 부모만 할 수 있는 것과 아이도 할 수 있는 것을 구분
- 포인트를 직접 수정하는 건 불가능하게 (적립/차감 기록만 추가)

자동 계산:
- 포인트가 쌓이면 레벨이 자동으로 올라가게
- 매일 할일이 자동으로 생성되게

Supabase SQL로 만들어줘!
```

## 프롬프트 #4 — 로그인 만들기 🔐

```
3단계야!

로그인/회원가입 화면을 만들어줘.

로그인 방법 2가지:
- 가족이름 + 내 이름 + 비밀번호 (아이들이 쓰기 쉽게)
- 이메일 + 비밀번호 (부모용)

회원가입:
- 부모가 이메일로 가입 → 이메일 인증
- 가족 만들기: 가족이름 + 초대코드 + 시간대 + 언어 선택
- 아이 가입: 가족이름 + 초대코드 + 이름 + 비밀번호 (이메일 필요 없음!)

첫 가입 후에 캐릭터를 고르는 화면도 만들어줘.
로그인 안 한 사람이 앱 화면에 들어가면 자동으로 로그인 페이지로 보내줘.
로그인 전 화면에 언어 선택 (🇰🇷/🇯🇵/🇺🇸) 넣어줘.
모든 텍스트는 하드코딩하지 말고 번역 파일(i18n)로 관리해줘.
```

## 프롬프트 #5 — 할일 관리 만들기 ✅

```
4단계야!

부모가 할일을 관리하는 화면을 만들어줘.

할일 등록:
- 제목, 포인트, 담당 아이 선택
- "매주 반복" 또는 "한번만" 선택
  - 반복이면: 요일 선택 (월화수목금토일 버튼)
  - 한번이면: 마감일 선택
- 등록된 할일 목록을 아이별로 보여줘

할일 기록:
- 날짜별로 과거 할일을 볼 수 있는 화면
- 부모는 체크/해제 수정 가능 (실수 수정용)
- 아이는 읽기 전용

할일 체크 컴포넌트는 여러 화면에서 공통으로 쓸 수 있게 만들어줘.
완료는 초록, 미완료는 회색, 조르기 성공은 보라 같은 식으로 색깔로 구분!
```

## 프롬프트 #6 — 아이 화면 + 성장 시스템 🌟

```
5단계야!

아이 화면과 포인트/캐릭터 시스템을 만들어줘.

아이 홈 화면:
- 오늘 할일 목록 + 체크 버튼
- 내 캐릭터 + 레벨 + 포인트

부모 홈 화면:
- 아이별 오늘 진행률 (몇 개 완료했는지 바 그래프)
- 각 할일 상태 표시

포인트 화면:
- 최근 포인트 내역 (언제 얼마 받았는지)

캐릭터 화면:
- 내 캐릭터 성장 단계 보기
  🐣 병아리(L1~6) → 🧒 루키(L7~12) → ⚔️ 용사(L13~18) → 🦸 영웅(L19~24) → 👑 전설(L25~30)
- 배지 도감: 획득한 배지는 크게, 아직 못 받은 건 회색으로
- 레벨별 필요 포인트 표

보상 화면:
- 보상 목록 + "갖고 싶어!" 버튼
- 부모가 승인하면 포인트 차감 + 보상 지급

레벨 계산은 DB와 앱 코드에서 똑같은 공식을 써야 해!
```

## 프롬프트 #7 — 조르기 기능 만들기 🙏

```
6단계야!

"조르기" 기능을 만들어줘. 아이가 "나 이것도 했어!" 하고 포인트를 요청하는 거야.

아이 화면:
- 할일 페이지 아래에 "오늘 내가 한 일" 입력칸
- 보내면 부모한테 요청이 감
- 대기 중인 요청은 목록으로 보여주고, 취소도 가능

부모 화면:
- 할일 페이지 위에 "조르기 요청" 카드
- 포인트를 입력하고 "승인" 또는 "거절"

승인되면 포인트가 바로 적립돼!
모든 화면에서 조르기 결과를 "🎉 조르기 성공!" / "❌ 조르기 실패" 로 보여줘.
```

## 프롬프트 #8 — 알림 + 앱 설치 📱

```
7단계야!

푸시 알림과 앱 설치 기능을 만들어줘.

앱 설치 (PWA):
- 스마트폰에서 "홈 화면에 추가"하면 진짜 앱처럼 보이게
- 앱 아이콘, 이름, 전체 화면 모드 설정

푸시 알림:
- 앱에 들어오면 자동으로 알림 허용을 물어봐
- 조르기 요청하면 → 부모한테 알림
- 조르기 승인/거절하면 → 아이한테 알림
- 보상 승인/거절하면 → 아이한테 알림
- 매일 저녁 9시 → 아직 할일 남은 아이한테 알림
```

## 프롬프트 #9 — 자동화 ⏰

```
8단계야!

매일 자동으로 실행되는 작업을 만들어줘.

매일 저녁 9시:
- 아직 안 끝낸 할일이 있는 아이한테 푸시 알림 보내기

매일 새벽 1시:
- 어제 안 한 할일은 "놓침" 처리 + 벌점 -50포인트
- 오늘 할일 자동 생성 (반복 할일 기준)

이 작업들은 아무도 앱에 안 들어와 있어도 서버에서 혼자 실행돼야 해.
비밀 키로 보호해서 외부에서 함부로 실행 못하게 해줘.
```

## 프롬프트 #11 — 인터넷에 올리기 🚀

> **부모님께:** 이 단계에서 [Vercel](https://vercel.com)과 Supabase 설정이 필요합니다. README의 "배포하기" 섹션을 참고하세요.

```
마지막 단계야!

만든 앱을 인터넷에 올려서 가족 모두가 쓸 수 있게 해줘.

Vercel로 배포하는 방법을 알려줘:
1. Vercel 연결하기
2. 환경변수 설정하기
3. Supabase에서 우리 앱 주소 등록하기
4. 배포하기!

배포 후 확인할 것:
- 가입이 되는지
- 할일 추가/체크가 되는지
- 포인트가 쌓이는지
- 푸시 알림이 오는지

끝! 🎉 우리 가족 앱 완성!
```
